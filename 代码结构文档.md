# Zircon MMORPG 游戏引擎 — 代码结构详细文档

## 一、项目概述

Zircon 是一个完整的 MMORPG（大型多人在线角色扮演游戏）引擎，基于 **C# / .NET** 平台开发，
是经典游戏《热血传奇3》(Legend of Mir 3) 的开源重制版本（Fork 自 Suprcode/Zircon）。

### 核心特性
- **4 个职业**：战士(Warrior)、法师(Wizard)、道士(Taoist)、刺客(Assassin)
- **7 种元素**：火(Fire)、冰(Ice)、雷(Lightning)、风(Wind)、圣(Holy)、暗(Dark)、幻(Phantom)
- **200+ 种技能**：每个职业 50+ 种技能，含修炼(Discipline)子系统
- **双渲染管线**：DirectX 9 和 DirectX 11（通过 SharpDX 库）
- **自定义数据库**：MirDB 二进制数据库系统
- **插件系统**：通过 PluginCore 实现服务端功能扩展
- **多语言支持**：英文/中文界面
- **Sentry 错误追踪**：客户端崩溃报告集成

### 技术栈
| 组件 | 技术 |
|------|------|
| 语言 | C# (.NET) |
| IDE | Visual Studio 2022 (17.4+) |
| 渲染 | SharpDX (DirectX 9 / DirectX 11) |
| UI 框架 | 自定义 DXControl 体系（基于 DirectX 绘制） |
| 服务端 UI | DevExpress WinForms (RibbonForm) |
| 数据库 | MirDB（自定义二进制序列化数据库） |
| 网络 | TCP 异步通信（TcpClient + 自定义协议） |
| 音频 | 自定义 DXSoundManager |
| 加密 | AES-256（可选数据库加密） |
| 错误追踪 | Sentry SDK |

```

├── Client/                  # 游戏客户端

├── ImageManager/            # 图片管理工具

```

### 3.1 Client（游戏客户端）

客户端是一个基于 DirectX 的 2D MMORPG 游戏客户端，采用**场景-控件-模型**三层架构。

#### 3.1.1 入口与初始化

| 文件 | 说明 |
|------|------|
| `Program.cs` | 应用程序主入口，负责：加载配置 → 加载图库 → 初始化渲染管线 → 启动主循环 |
| `TargetForm.cs` | 游戏主窗口（继承 `SharpDX.Windows.RenderForm`），转发鼠标/键盘事件到场景，管理截图功能 |

**启动流程**：
```
Program.Main()
  → ConfigReader.Load()          // 读取 Zircon.ini 配置文件
  → Sentry 初始化（可选）
  → Program.Init()
    → Application 基础设置
    → 加载所有 .Zl 图库文件（MirLibrary）
    → CEnvir.Init()              // 初始化客户端环境
    → 创建 TargetForm            // 创建 DirectX 渲染窗口
    → RenderingPipelineManager   // 初始化渲染管线（DX9/DX11 自动回退）
    → DXSoundManager.Create()    // 初始化音效系统
    → LoginScene                 // 创建登录场景
    → RunMessageLoop()           // 启动游戏主循环
  → ConfigReader.Save()          // 退出时保存配置
```

**通信协议**：
- 基于 TCP 的自定义二进制协议
- 数据包格式：`[长度:4字节][类型ID:2字节][数据:N字节]`
- 通过反射自动注册/发现数据包类型
- 支持异步发送和接收，线程安全队列

## 四、核心系统架构

### 4.1 网络通信架构

```
┌─────────────┐                          ┌─────────────┐
│   Client     │                          │   Server     │
│              │                          │              │
│ CConnection  │◄─── TCP 异步通信 ───►│ SConnection  │
│   (Client)   │                          │   (Server)   │
│              │                          │              │
│ Process()    │  ClientPackets (C→S)    │ Process()    │
│ 处理 S→C 包  │ ──────────────────── │ 处理 C→S 包  │
│              │  ServerPackets (S→C)    │              │
│              │ ◄──────────────────── │              │
└──────┬──────┘                          └──────┬──────┘
       │                                        │
  BaseConnection (共享基类)               BaseConnection
  ├── 异步接收 (BeginReceive)             ├── 异步接收
  ├── 发送队列 (SendList)                 ├── 发送队列
  ├── 接收队列 (ReceiveList)              ├── 接收队列
  └── 超时检测                            └── 超时检测
```

### 4.2 渲染架构

```
RenderingPipelineManager (静态管理器)
  │
  ├── 初始化: InitializeWithFallback()
  │   → 尝试 DX11 → 失败则回退 DX9
  │
  ├── 主循环: RunMessageLoop()
  │   → 每帧调用 GameLoop()
  │     → UpdateGame()  // 逻辑更新
  │     → RenderGame()  // 渲染帧
  │       → RenderFrame(() => ActiveScene.Draw())
  │
  ├── 渲染操作:
  │   ├── DrawTexture()    // 绘制纹理
  │   ├── DrawLine()       // 绘制线条
  │   ├── SetOpacity()     // 设置透明度
  │   ├── SetBlend()       // 设置混合模式
  │   ├── CreateTexture()  // 创建 GPU 纹理
  │   └── ...
  │
  └── 实现:
      ├── SharpDXD3D9RenderingPipeline
      └── SharpDXD3D11RenderingPipeline
```

### 4.3 场景-控件架构

```
DXControl (基类)
  ├── 静态属性: MouseControl, FocusControl, ActiveScene
  │
  └── DXScene (场景基类)
      ├── LoginScene (登录场景)

```

## 五、关键数据流

### 5.1 登录流程

```
客户端:                              服务端:
  LoginDialog 输入邮箱/密码
  → C.Login {EMail, Password} ─────→ SConnection.Process(C.Login)
                                       → 验证账号密码
                                       → 创建/获取 AccountInfo
  ← S.NewAccount {Result} ──────── ← → 返回结果
  SelectScene 选择角色
  → C.StartGame {CharIndex} ──────→ SConnection.Process(C.StartGame)
                                       → 创建 PlayerObject
                                       → 加载角色数据
                                       → 发送地图/物品/技能等数据
  ← S.StartGame {成功} ─────────── ←
  GameScene 进入游戏
```

### 5.2 游戏主循环

```
客户端 (60 FPS):                    服务端 (50 Tick/s):
  GameLoop()                        EnvirLoop()
  ├── UpdateGame()                  ├── 处理新连接
  │   ├── Now = Time.Now            ├── 遍历 ActiveObjects
  │   ├── Connection.Process()      │   └── object.Process()
  │   │   └── 处理接收队列           │       ├── 移动/攻击/Buff
  │   ├── ActiveScene.Process()     │       ├── AI 行为
  │   │   └── 场景逻辑更新          │       └── 法术效果
  │   └── 更新调试信息              ├── 处理延迟动作
  └── RenderGame()                  ├── 定期保存
      └── ActiveScene.Draw()        └── 日志输出
```

## 七、资源文件格式

### 7.1 图库文件 (.Zl)

自定义的二进制精灵图库格式：
- 头部：图像数量 + 版本号
- 每个图像：偏移位置、宽高、偏移量、阴影类型
- 像素数据：DXT1 (v0) 或 DXT5 (v1+) 压缩格式
- 支持三层：Image（主图）、Shadow（阴影）、Overlay（叠加层）

### 7.2 地图文件 (.map)

地图数据文件，定义地形、阻挡、光照等信息。

## 八、扩展与定制

### 8.1 添加新技能
1. 在 `LibraryCore/Enum.cs` 的 `MagicType` 枚举中添加新技能 ID
2. 在 `ServerLibrary/Models/Magics/{职业}/` 下创建技能实现类
3. 在 `LibraryCore/Functions.cs` 中添加动画映射
4. 在数据库中配置 `MagicInfo` 记录

### 8.2 添加新怪物
1. 在 `LibraryCore/Enum.cs` 的 `MonsterImage` 添加怪物图像类型
2. 在数据库中配置 `MonsterInfo` + `RespawnInfo` + `DropInfo`
3. 如需特殊 AI，在 `ServerLibrary/Models/Monsters/` 创建子类

### 8.3 添加新插件
1. 创建 `Plugin.{名称}` 项目
2. 实现 `IPluginStart` 和 `AbstractPlugin`
3. 编译为 `Plugin.{名称}.dll` 放到服务端目录
